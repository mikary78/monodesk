# MonoDesk PRD (Product Requirements Document)
# 제품 요구사항 문서

문서 버전: v1.0
작성일: 2025년
작성자: MonoBound 대표 (Claude Code 협업)

---

## 1. 프로젝트 개요

### 1-1. 제품 정의
MonoDesk는 용산 삼각지 제철해산물 전문 주점 **"여남동"** 의
법인 운영 전반을 관리하는 **로컬 기반 통합 경영 관리 웹앱**이다.

### 1-2. 핵심 목표
- 외식업 운영에 필요한 모든 관리 업무를 하나의 프로그램으로 통합
- 세무사 제출 자료를 자동으로 정리하여 업무 시간 단축
- POS 데이터 기반 매출 분석으로 데이터 기반 경영 실현
- AI 기능으로 반복 업무(영수증 입력, 회의록 정리) 자동화
- 법인 운영에 필요한 문서와 일정을 체계적으로 관리

### 1-3. 사용자 정의
```
주 사용자  : MonoBound 법인 대표 (1인)
사용 목적  : 매장 운영 전반 관리 및 경영 의사결정 지원
사용 환경  : Windows PC (로컬 전용)
사용 빈도  : 매일 (매출 확인, 지출 입력 등)
기술 수준  : 코딩 비전문가 (바이브코딩 경험 보유)
```

### 1-4. 사업체 정보
```
법인명     : MonoBound
매장명     : 여남동
업종       : 제철해산물 전문 주점
위치       : 용산구 삼각지
동업자     : 총 4명 (3명 각 29%, 1명 13%)
정산 방식  : 급여(매월) + 배당(연말 결산 후)
세무 처리  : 세무사 위탁 (자료 준비는 직접)
POS       : 미정 (CSV/Excel 데이터 추출 가능한 기종 선택 예정)
```

## 2. 기술 스택

```
Frontend   : React 18 + Tailwind CSS
Backend    : Python FastAPI
Database   : SQLite + SQLAlchemy
AI (로컬)  : Ollama (llama3.2, llava) + faster-whisper + Tesseract + ChromaDB
연결 브릿지 : LangChain
개발 도구  : VS Code + Claude Code
실행 환경  : Windows 로컬 (브라우저로 접속)
```

## 3. 모듈 정의 및 상세 요구사항

### 모듈 개발 우선순위
```
1순위 → 세무/회계 관리
2순위 → 매출 분석 (POS 연동)
3순위 → 재고/발주 관리
4순위 → 메뉴 관리
5순위 → 직원 관리
6순위 → 대시보드
7순위 → 법인 관리
8순위 → 운영 관리
```

---

### 모듈 1. 세무/회계 관리 ⭐ 1순위

#### 목적
세무사에게 넘길 자료를 자동 정리하고, 대표님이 매월 손익을 직접 파악한다.

#### 핵심 기능

**1-1. 매출 입력 및 관리**
- 일별 매출 수동 입력 (카드 / 현금 / 기타)
- POS CSV 데이터 가져오기 (자동 입력)
- 월별 매출 합계 자동 계산
- 매출 수정 및 삭제 (소프트 삭제)

**1-2. 지출 입력 및 관리**
- 지출 항목 직접 입력
- 지출 분류 (재료비 / 임대료 / 인건비 / 공과금 / 기타)
- 영수증 이미지 업로드 → AI OCR 자동 인식
- 세금계산서 수취 여부 체크
- 원본 이미지 보관

**1-3. 월별 손익 현황**
- 총 매출 / 총 지출 / 순이익 자동 계산
- 항목별 지출 비중 차트
- 전월 대비 증감률 표시
- 원가율 자동 계산 (재료비 ÷ 매출)

**1-4. 동업자 정산 시뮬레이션**
- 4명 지분 비율 설정 (3명 각 29%, 1명 13%)
- 월별 순이익 기반 지분 배분 금액 자동 계산
- 급여 지급 내역 차감 후 잔여 배당 가능 금액 표시
- 연간 누적 정산 현황

**1-5. 세무사 제출용 리포트**
- 월간 매출/지출 내역 Excel 자동 생성
- 항목별 분류 정리
- 파일명 자동 지정 (report_YYYY_MM.xlsx)

#### 화면 구성
```
[세무/회계]
├── 매출 관리     → 일별 매출 입력 + 목록
├── 지출 관리     → 지출 입력 + 영수증 업로드 + 목록
├── 손익 현황     → 월별 손익 차트 + 요약
├── 지분 정산     → 동업자별 배분 시뮬레이션
└── 리포트 출력   → 세무사 제출용 Excel 생성
```

### 모듈 2. 매출 분석 ⭐ 2순위

#### 목적
POS 데이터를 분석하여 메뉴 전략과 운영 전략 수립에 활용한다.

#### 핵심 기능

**2-1. POS 데이터 연동**
- CSV / Excel 파일 가져오기
- 데이터 미리보기 후 가져오기 확인
- 중복 데이터 감지 및 처리

**2-2. 매출 트렌드 분석**
- 일별 / 주별 / 월별 / 계절별 매출 추이
- 전기간 대비 증감률
- 목표 매출 대비 달성률

**2-3. 메뉴 분석**
- 메뉴별 판매량 및 매출 기여도
- 인기 메뉴 TOP 10 / 비인기 메뉴 목록
- 계절별 메뉴 판매 패턴 (제철 해산물 특성 반영)
- 메뉴별 마진율 분석

**2-4. 시간대/요일 분석**
- 요일별 매출 패턴
- 시간대별 피크 분석
- 한산한 시간대 프로모션 제안

**2-5. 결제 수단 분석**
- 카드 / 현금 / 네이버페이 등 비중
- 월별 결제 수단 변화 추이

**2-6. AI 경영 인사이트**
- Ollama 기반 자연어 분석 리포트 자동 생성
- 매출 이상 감지 및 알림
- 재고 소진 예측 및 발주 추천

---

### 모듈 3. 재고/발주 관리 ⭐ 3순위

#### 목적
식재료 재고를 실시간으로 파악하고 발주를 효율적으로 관리한다.

#### 핵심 기능
- 식재료 재고 현황 등록 및 관리
- 거래처(납품업체) 정보 관리
- 발주 내역 기록 및 이력 관리
- 매출 데이터 연동 재고 소진 예측
- 재고 부족 시 대시보드 알림
- 원가율 자동 계산

---

### 모듈 4. 메뉴 관리 ⭐ 4순위

#### 목적
메뉴 레시피와 원가를 체계적으로 관리하고 메뉴 개발을 지원한다.

#### 핵심 기능
- 메뉴 정보 등록 (이름, 카테고리, 가격, 사진)
- 메뉴별 레시피 등록 (재료 + 수량)
- 원가 / 단가 / 마진율 자동 계산
- 계절 메뉴 관리 (제철 해산물 특성 반영)
- 가격 변경 이력 관리
- 메뉴 개발 아이디어 노트

---

### 모듈 5. 직원 관리 ⭐ 5순위

#### 목적
정직원과 아르바이트생을 통합 관리하고 급여를 자동 계산한다.

#### 핵심 기능
- 직원 정보 등록 (정직원 / 아르바이트 구분)
- 근태 기록 (출퇴근 시간)
- 급여 자동 계산 (시급 / 월급)
- 4대보험 공제 내역 관리
- 근로계약서 파일 보관
- 급여 지급 이력 관리

---

### 모듈 6. 대시보드 ⭐ 6순위

#### 목적
매일 아침 한 화면으로 매장 현황을 파악한다.

#### 핵심 기능
- 오늘 / 이번 주 / 이번 달 매출 KPI 카드
- 매출 트렌드 미니 차트
- 주요 일정 알림 (급여일, 세금납부일, 계약만료일 등)
- 재고 부족 알림
- 인기 메뉴 TOP 5
- 최근 지출 내역
- AI 경영 인사이트 요약

---

### 모듈 7. 법인 관리 ⭐ 7순위

#### 목적
법인 운영에 필요한 문서와 일정을 체계적으로 관리한다.

#### 핵심 기능

**7-1. 주주총회 회의록 관리**
- 음성 파일 업로드 → Whisper 텍스트 변환
- Ollama 기반 회의록 자동 정리 및 양식 생성
- 회의록 검토 / 수정 / 저장
- 회의록 이력 관리 및 검색

**7-2. 법인 문서 관리**
- 법인 정관 보관
- 등기 이력 관리
- 각종 허가증 보관 및 갱신 일정 알림

**7-3. 계약서 관리**
- 임대차 계약서
- 거래처 계약서
- 계약 만료일 자동 알림

**7-4. 법정 의무 일정 캘린더**
- 법인세 신고 (3월)
- 부가세 신고 (1월, 7월)
- 주주총회 (매년)
- 각종 허가 갱신 일정

---

### 모듈 8. 운영 관리 ⭐ 8순위

#### 목적
매일 반복되는 운영 업무를 체크리스트로 관리한다.

#### 핵심 기능
- 오픈 / 마감 체크리스트
- 위생 점검 일지 (식품위생법 기준)
- 영업일지 작성
- 시설 고장 / 수리 이력 관리

## 4. AI 기능 요구사항

### 4-1. 영수증 OCR
```
입력  : 영수증 이미지 (인쇄체 / 수기)
처리  : Tesseract(인쇄체) + Ollama llava(수기)
출력  : 날짜 / 거래처 / 품목 / 금액 자동 추출
조건  : 사용자 확인 후 저장 (자동 저장 금지)
```

### 4-2. 회의록 자동 정리
```
입력  : 음성 파일 (mp3 / wav / m4a)
처리  : faster-whisper(텍스트변환) → Ollama(정리)
출력  : 주주총회 회의록 양식으로 자동 생성
조건  : 사용자 검토 / 수정 후 최종 저장
```

### 4-3. 매출 분석 리포트
```
입력  : POS 데이터 + 지출 데이터
처리  : Ollama llama3.2 분석
출력  : 자연어 경영 인사이트 리포트
언어  : 한국어
```

### 4-4. 문서 자연어 검색
```
입력  : 자연어 검색어 (예: "임대차 계약 만료일")
처리  : ChromaDB 벡터 검색
출력  : 관련 문서 TOP 3 + 출처 표시
```

## 5. 비기능 요구사항

### 5-1. 성능
- 화면 로딩: 3초 이내
- 데이터 저장: 1초 이내
- AI 처리: 진행 상태 표시 (프로그레스 바)

### 5-2. 보안
- 모든 데이터 로컬 저장 (외부 서버 없음)
- 1등급 민감 데이터 암호화 저장
- 화면 내 민감 정보 마스킹 처리

### 5-3. 안정성
- 자동 백업: 매일 자정
- AI 없이도 모든 기본 기능 동작
- 에러 발생 시 한국어 안내 메시지

### 5-4. 사용성
- 한국어 인터페이스 100%
- 클릭 3번 이내로 주요 기능 접근
- 모바일 추후 확장 고려한 구조
