## 1. 프로그램 시작 방법

MonoDesk는 **서버 하나만 실행**하면 바로 사용할 수 있습니다.

### 1-1. 원클릭 실행 (권장)

1. Windows 파일 탐색기에서 MonoDesk 폴더를 엽니다.
2. **`start.bat`** 파일을 **더블클릭**합니다.
3. 검은 창(명령 프롬프트)이 열리면서 서버가 시작됩니다.
4. 브라우저가 자동으로 열리며 MonoDesk 화면이 나타납니다.
5. 이 창은 닫지 마세요. 프로그램을 사용하는 동안 계속 켜둬야 합니다.

> **처음 실행할 때**는 프론트엔드 빌드가 자동으로 진행됩니다. 약 1~2분 소요되며 이후에는 즉시 실행됩니다.

### 1-2. 수동 실행 (원클릭이 안 될 때)

1. **명령 프롬프트(cmd)** 또는 **PowerShell**을 엽니다.
2. MonoDesk 폴더 안의 `backend` 폴더로 이동합니다.

   ```
   cd C:\경로\MonoDesk\backend
   ```

3. 아래 명령어를 입력하고 Enter를 누릅니다.

   ```bash
   python -m uvicorn main:app --host 0.0.0.0 --port 8000
   ```

4. 화면에 `MonoDesk 서버가 시작되었습니다.` 라고 표시되면 성공입니다.

### 1-3. 브라우저에서 열기

브라우저가 자동으로 열리지 않는 경우, **Chrome** 또는 **Edge** 주소창에 아래를 입력합니다.

```
http://localhost:8000
```

> **주의**: 검은 창(서버 창)이 꺼져 있으면 데이터가 불러와지지 않습니다. 항상 서버 창을 켜둔 상태로 사용하세요.

## 2. 대시보드

화면 왼쪽 메뉴에서 **대시보드**를 클릭하면 진입합니다.

### 주요 구성

- **KPI 카드**: 이번 달 매출, 지출, 순이익, 직원 수 등 핵심 지표를 한눈에 확인합니다.
- **매출 트렌드 차트**: 최근 기간의 매출 추이를 그래프로 보여줍니다.
- **재고 알림**: 최소 재고 기준 이하로 떨어진 품목이 표시됩니다.
- **급여 현황**: 이번 달 급여 지급 예정 금액을 표시합니다.

대시보드는 읽기 전용입니다. 데이터 입력은 각 모듈에서 합니다.

## 3. 세무/회계 관리

왼쪽 메뉴에서 **세무/회계**를 클릭합니다. 5개의 탭으로 구성되어 있습니다.

### 탭 1: 매출

매일의 매출을 기록합니다.

**입력 방법**
1. 오른쪽 위 **매출 추가** 버튼을 클릭합니다.
2. 날짜, 카드 매출, 현금 매출, 기타 매출을 입력합니다.
3. **저장** 버튼을 클릭합니다.

**수정/삭제**
- 목록에서 해당 행의 수정(연필) 아이콘 또는 삭제(휴지통) 아이콘을 클릭합니다.

### 탭 2: 지출

지출 내역을 기록합니다.

**입력 방법**
1. **지출 추가** 버튼을 클릭합니다.
2. 날짜, 분류(재료비·임대료·인건비 등), 거래처명, 내용, 금액을 입력합니다.
3. 세금계산서가 있는 경우 체크박스에 표시합니다.
4. 영수증 이미지를 첨부할 수 있습니다.
5. **저장**을 클릭합니다.

### 탭 3: 손익

월별 손익 계산서를 자동으로 계산하여 보여줍니다.

- 상단에서 **연도**와 **월**을 선택하면 해당 월의 손익이 표시됩니다.
- 총매출 - 총지출 = 순이익이 자동 계산됩니다.

### 탭 4: 지분 정산

동업자 4인의 지분(29% × 3명, 13% × 1명)에 따른 정산 금액을 시뮬레이션합니다.

- 연도와 기준 금액을 선택하면 각 동업자별 정산 예상 금액이 표시됩니다.
- 실제 정산은 세무사와 협의 후 진행하세요.

### 탭 5: 리포트

월별 Excel 리포트를 생성합니다.

1. 연도와 월을 선택합니다.
2. **리포트 생성** 버튼을 클릭합니다.
3. Excel 파일이 다운로드됩니다.

## 4. 매출 분석

왼쪽 메뉴에서 **매출 분석**을 클릭합니다. 6개의 탭으로 구성됩니다.

### 탭 1: POS 연동

POS에서 내보낸 데이터 파일을 가져옵니다.

1. POS 시스템에서 CSV 또는 Excel 형식으로 매출 데이터를 내보냅니다.
2. **파일 가져오기** 버튼을 클릭하고 파일을 선택합니다.
3. 가져온 데이터를 확인하고 **확인** 버튼을 클릭합니다.

### 탭 2: 트렌드

기간별 매출 트렌드를 차트로 확인합니다.

- 일별, 주별, 월별로 구분하여 볼 수 있습니다.
- 조회 기간을 선택하여 원하는 구간의 데이터를 확인합니다.

### 탭 3: 메뉴

메뉴별 판매량과 매출 기여도를 분석합니다.

- 어떤 메뉴가 가장 많이 팔리는지 확인할 수 있습니다.

### 탭 4: 시간대

시간대별 매출 분포를 확인합니다.

- 피크 타임과 한산한 시간대를 파악할 수 있습니다.

### 탭 5: 결제수단

카드, 현금 등 결제수단별 매출 비중을 확인합니다.

### 탭 6: AI 인사이트

AI가 매출 데이터를 분석하여 인사이트를 제공합니다.

> **참고**: AI 인사이트 기능은 Ollama가 설치되어 있어야 작동합니다. (추후 구현 예정)

## 5. 재고/발주 관리

왼쪽 메뉴에서 **재고/발주**를 클릭합니다. 3개의 탭으로 구성됩니다.

### 탭 1: 품목 관리

재고 항목(식재료 등)을 관리합니다.

**새 품목 추가**
1. **품목 추가** 버튼을 클릭합니다.
2. 재료명, 단위(kg/L/개 등), 현재 재고량, 최소 재고량, 단가, 주거래처를 입력합니다.
3. **저장**을 클릭합니다.

> **최소 재고량**: 이 값 이하로 떨어지면 대시보드에 알림이 표시됩니다. 적절한 수량을 설정하세요.

**재고 수량 수정**
- 수정 아이콘을 클릭하여 현재 재고량을 수정합니다.

### 탭 2: 발주 관리

거래처에 발주를 등록합니다.

**발주 등록**
1. **발주 등록** 버튼을 클릭합니다.
2. 발주일, 거래처, 품목, 수량, 단가를 입력합니다.
3. **저장**을 클릭합니다.

**입고 처리**
- 발주 목록에서 물건이 도착하면 **입고 완료** 버튼을 클릭합니다.
- 입고 처리 시 재고가 자동으로 증가합니다.

### 탭 3: 조정 이력

재고 조정 내역을 확인합니다.

- 언제, 어떤 항목의 재고가 얼마나 변경되었는지 이력을 조회합니다.

## 6. 메뉴 관리

왼쪽 메뉴에서 **메뉴 관리**를 클릭합니다.

### 메뉴 추가

1. **메뉴 추가** 버튼을 클릭합니다.
2. 메뉴명, 카테고리, 판매가, 계절 구분(연중/봄/여름/가을/겨울), 설명을 입력합니다.
3. **저장**을 클릭합니다.

### 레시피 등록 (원가 계산용)

1. 메뉴 목록에서 메뉴를 선택합니다.
2. **레시피 편집** 버튼을 클릭합니다.
3. 사용되는 재료, 수량, 단위, 단가를 입력합니다.
4. 레시피 재료를 저장하면 **원가**와 **마진율**이 자동 계산됩니다.

### 카테고리 관리

- 카테고리 탭에서 메뉴 분류를 추가하거나 수정합니다.

### 원가 분석

- 원가 분석 탭에서 메뉴별 마진율과 수익성을 한눈에 확인합니다.

## 7. 직원 관리

왼쪽 메뉴에서 **직원 관리**를 클릭합니다. 3개의 탭으로 구성됩니다.

### 탭 1: 직원 정보

**직원 등록**
1. **직원 추가** 버튼을 클릭합니다.
2. 이름, 구분(정직원/아르바이트), 직책, 입사일, 연락처, 계좌번호를 입력합니다.
3. 정직원은 **월급**, 아르바이트는 **시급**을 입력합니다.
4. **저장**을 클릭합니다.

> **주의**: 연락처와 계좌번호는 개인정보입니다. 화면에서 마스킹(***) 처리되어 표시됩니다.

**퇴사 처리**
- 직원 정보에서 **퇴사 처리** 버튼을 클릭하고 퇴사일을 입력합니다.

### 탭 2: 근태 관리

**근태 기록 입력**
1. **근태 추가** 버튼을 클릭합니다.
2. 직원, 날짜, 출근 시간, 퇴근 시간을 입력합니다.
3. 근무 시간이 자동으로 계산됩니다.

### 탭 3: 급여 관리

**급여 계산**
1. 연도와 월을 선택합니다.
2. **급여 계산** 버튼을 클릭합니다.
3. 직원별 기본급, 근태 기반 수당, 공제액(4대보험), 실수령액이 자동 계산됩니다.

**급여 지급 확인**
- 지급이 완료되면 해당 직원의 급여 행에서 **지급 완료** 표시를 합니다.

## 8. 법인 관리

왼쪽 메뉴에서 **법인 관리**를 클릭합니다.

### 동업자 정보

4인 동업 구조(29% × 3명, 13% × 1명)가 등록되어 있습니다.

- 각 동업자의 이름, 지분율, 역할을 확인합니다.

### 배당 정산

연말 배당 시 지분에 따른 배당금을 계산합니다.

1. 배당 연도와 기준 순이익을 입력합니다.
2. 동업자별 배당금, 배당소득세, 실수령액이 자동 계산됩니다.

> **참고**: 실제 배당 집행은 반드시 세무사와 협의 후 진행하세요.

### 법인 비용

법인 카드 사용 내역 등 법인 비용을 기록합니다.

## 9. 운영 관리

왼쪽 메뉴에서 **운영 관리**를 클릭합니다. 4개의 탭으로 구성됩니다.

### 탭 1: 공지사항

직원들에게 전달할 공지사항을 작성하고 관리합니다.

### 탭 2: 위생 점검

위생 점검 항목을 체크리스트로 관리합니다.

- 점검 날짜, 항목별 합격/불합격 여부, 메모를 기록합니다.

### 탭 3: 영업 캘린더

월별 영업 일정을 캘린더 형식으로 관리합니다.

- 휴무일, 이벤트, 특별 영업일 등을 등록합니다.

### 탭 4: 업무 체크리스트

반복되는 일일/주간 업무를 체크리스트로 관리합니다.

- 마감 점검, 청소, 재고 확인 등 루틴 업무를 등록하고 완료 여부를 기록합니다.

## 10. 데이터 백업 방법

MonoDesk의 모든 데이터는 아래 파일에 저장됩니다.

```
MonoDesk/database/monodesk.db
```

### 수동 백업 방법

1. Windows 파일 탐색기를 엽니다.
2. `MonoDesk\database\` 폴더로 이동합니다.
3. `monodesk.db` 파일을 복사합니다.
4. `MonoDesk\backup\manual\` 폴더에 붙여넣습니다.
5. 파일 이름에 날짜를 추가합니다. 예: `monodesk_20250301.db`

### 백업 권장 주기

- **매일**: 영업 종료 후 수동 백업
- **주 1회**: 외부 드라이브(USB 등)에 추가 백업
- **월 1회**: 클라우드 저장소(OneDrive 등)에 백업

### 복원 방법

데이터를 이전 상태로 되돌리려면:

1. 백엔드 서버를 종료합니다 (명령 프롬프트에서 Ctrl+C).
2. `MonoDesk\database\monodesk.db` 파일을 삭제합니다.
3. 복원하려는 백업 파일을 복사하여 `monodesk.db` 이름으로 붙여넣습니다.
4. 백엔드 서버를 다시 시작합니다.

## 11. 자주 묻는 질문 (FAQ)

### Q. 프로그램이 시작되지 않아요.

**A.** 아래를 순서대로 확인하세요.

1. **`start.bat`을 실행했는지 확인하세요.** MonoDesk 폴더에서 `start.bat`을 더블클릭합니다.
2. **서버 창(검은 창)이 열려 있는지 확인하세요.** 창이 닫혀 있으면 서버가 꺼진 것입니다.
3. 브라우저 주소창에 `http://localhost:8000` 을 다시 입력해보세요.

---

### Q. 데이터가 저장되지 않아요.

**A.** 서버가 꺼져 있을 가능성이 높습니다. `start.bat`을 실행 중인 검은 창을 확인하세요. 오류 메시지가 있다면 창을 닫고 `start.bat`을 다시 실행하세요.

---

### Q. 화면이 이상하게 보여요.

**A.** 브라우저를 새로고침(F5)해보세요. 그래도 안 되면 브라우저 캐시를 지운 후 다시 시도하세요 (Ctrl+Shift+Delete).

---

### Q. 실수로 데이터를 삭제했어요.

**A.** 백업 파일이 있다면 [복원 방법](#복원-방법) 섹션을 참고하여 복원하세요. 백업이 없다면 복구가 어렵습니다. 정기적인 백업을 습관화하세요.

---

### Q. 직원 연락처나 계좌번호가 ***로 표시돼요.

**A.** 정상입니다. 개인정보 보호를 위해 민감한 정보는 화면에서 마스킹 처리됩니다. 수정이 필요하면 수정 버튼을 클릭하면 입력 필드에서 전체 값을 확인하고 변경할 수 있습니다.

### Q. 실행 시 "포트가 이미 사용 중"이라는 오류가 납니다

**A.** 이미 다른 창에서 서버가 실행 중입니다. 모든 검은 창(명령 프롬프트)을 닫고 `start.bat`을 다시 실행하세요. 그래도 안 된다면 작업 관리자(Ctrl+Shift+Esc)에서 Python 프로세스를 종료한 후 재실행하세요.

---

### Q. AI 인사이트 기능이 작동하지 않아요.

**A.** AI 기능은 Ollama가 설치·실행 중이어야 합니다. 현재 버전에서는 AI 기능이 준비 중입니다. 추후 업데이트를 통해 제공될 예정입니다.

---

### Q. Excel 리포트 파일이 다운로드되지 않아요.

**A.** 브라우저의 다운로드 차단 설정을 확인하세요. Chrome의 경우 주소창 오른쪽의 다운로드 차단 알림을 클릭하여 허용하세요.
