# MonoDesk 데이터 관리 정책 (Data Policy)

이 문서는 MonoDesk에서 생성되고 저장되는 모든 데이터의 관리 기준입니다.
모든 데이터는 사용자 PC 로컬에만 저장되며, 외부로 절대 유출되지 않습니다.

---

## 1. 핵심 원칙

- **로컬 전용**: 모든 데이터는 사용자 PC 내에만 저장, 외부 서버/클라우드 저장 금지
- **데이터 보호**: 민감 데이터(급여, 지분, 재무)는 암호화 저장
- **보존 의무**: 세법 및 법인 운영 기준에 따른 보존 기간 준수
- **정기 백업**: 데이터 손실 방지를 위한 자동/수동 백업 운영
- **접근 통제**: 프로그램 내 데이터 접근 권한 명확히 구분

---

## 2. 데이터 분류 및 보안 등급

### 1등급 — 최민감 데이터 (암호화 저장 필수)
```
- 직원 개인정보 (이름, 주민번호, 계좌번호, 연락처)
- 직원 급여 및 4대보험 내역
- 동업자 지분 비율 및 정산 내역
- 법인 재무 데이터 (매출, 순이익, 배당)
- 법인 계좌 관련 정보
```

### 2등급 — 민감 데이터 (접근 주의)
```
- 일별/월별 매출 및 지출 데이터
- 거래처 정보 및 계약 내용
- 계약서, 법인 정관, 등기 서류
- 세무 신고 관련 자료
- 영수증 이미지 원본
```

### 3등급 — 일반 데이터 (일반 관리)
```
- 메뉴 정보 및 레시피
- 재고 현황 및 발주 내역
- 운영 체크리스트
- 영업일지
- 위생 점검 일지
```

## 3. 데이터 저장 구조

### 폴더 구조
```
MonoDesk/
├── database/
│   ├── monodesk.db          # SQLite 메인 데이터베이스
│   └── chromadb/            # ChromaDB 벡터 데이터 (AI 문서 검색용)
│
├── uploads/
│   ├── receipts/            # 영수증 이미지
│   │   └── YYYY/MM/         # 연월별 폴더 자동 생성
│   ├── documents/           # 법인 문서
│   │   ├── contracts/       # 계약서
│   │   ├── minutes/         # 회의록
│   │   └── legal/           # 법인 정관, 등기 서류
│   └── audio/               # 회의 음성 파일
│       └── YYYY/MM/
│
└── backup/
    ├── auto/                # 자동 백업 (일별)
    │   └── YYYY-MM-DD/
    └── manual/              # 수동 백업
        └── YYYYMMDD_HHMMSS/
```

### 파일 명명 규칙
```
영수증 이미지  : receipt_YYYYMMDD_거래처명.jpg
회의록 음성    : audio_YYYYMMDD_회의명.mp3
회의록 문서    : minutes_YYYYMMDD_회의명.md
계약서         : contract_YYYYMMDD_계약명.pdf
리포트         : report_YYYY_MM_항목명.xlsx
백업 파일      : backup_YYYYMMDD_HHMMSS.zip
```

## 4. 데이터베이스 설계 원칙

### 공통 규칙
```sql
-- 모든 테이블 필수 컬럼
id         INTEGER PRIMARY KEY AUTOINCREMENT,  -- 고유 식별자
created_at DATETIME DEFAULT CURRENT_TIMESTAMP, -- 생성 일시
updated_at DATETIME DEFAULT CURRENT_TIMESTAMP, -- 수정 일시
is_deleted INTEGER DEFAULT 0                   -- 소프트 삭제 (0: 정상, 1: 삭제)
```

### 소프트 삭제 원칙
- 데이터는 실제로 삭제하지 않고 `is_deleted = 1` 로 처리
- 실수로 삭제한 경우 복구 가능
- 완전 삭제는 대표님 명시적 요청 시에만 실행

### 주요 테이블 목록
```
sales_records        # 매출 기록
expense_records      # 지출 기록
expense_categories   # 지출 분류 항목
employees            # 직원 정보
salary_records       # 급여 지급 내역
attendance_records   # 근태 기록
menu_items           # 메뉴 정보
menu_recipes         # 메뉴 레시피
inventory_items      # 재고 항목
inventory_records    # 재고 입출고 기록
suppliers            # 거래처 정보
purchase_orders      # 발주 내역
shareholders         # 주주(동업자) 정보
dividend_records     # 배당 기록
corporate_minutes    # 주주총회 회의록
contracts            # 계약서 관리
receipt_images       # 영수증 이미지 메타데이터
```

## 5. 데이터 보존 기간

### 법적 기준 보존 기간
```
매출/지출 데이터      : 5년 (부가가치세법 기준)
영수증 이미지 원본    : 5년 (세법 증빙 서류 기준)
급여 지급 내역        : 5년 (소득세법 기준)
4대보험 관련 서류     : 3년 (사회보험법 기준)
근로계약서            : 퇴직 후 3년
법인 정관             : 영구 보존
주주총회 회의록       : 영구 보존
등기 관련 서류        : 영구 보존
계약서                : 계약 종료 후 5년
운영 일지             : 1년
위생 점검 일지        : 2년 (식품위생법 기준)
```

### 자동 보존 기간 알림
- 보존 기간 만료 30일 전 대시보드 알림
- 만료 데이터 목록 제공 후 대표님 확인 하에 삭제 여부 결정
- 영구 보존 항목은 자동 삭제 대상에서 제외

## 6. 백업 정책

### 자동 백업
```
주기     : 매일 자정 (00:00)
대상     : 전체 데이터베이스 + 업로드 파일
저장위치 : /backup/auto/YYYY-MM-DD/
보관기간 : 최근 30일 (30일 이전 자동 삭제)
형식     : ZIP 압축
```

### 수동 백업
```
위치     : 앱 내 [설정 → 백업 → 지금 백업] 버튼
저장위치 : /backup/manual/YYYYMMDD_HHMMSS/
보관기간 : 수동 삭제 전까지 영구 보관
권장시기 : 중대한 변경 전, 월말 결산 후
```

### 백업 복원
```
위치     : 앱 내 [설정 → 백업 → 복원]
방식     : 백업 날짜 선택 → 복원 미리보기 → 확인 후 복원
주의     : 복원 시 현재 데이터 덮어쓰기 → 복원 전 자동 백업 생성
```

### 백업 파일 구성
```
backup_YYYYMMDD_HHMMSS.zip
├── monodesk.db          # 메인 데이터베이스
├── chromadb/            # 벡터 데이터
├── uploads/             # 업로드 파일 전체
└── backup_info.json     # 백업 정보 (날짜, 버전, 데이터 건수)
```

## 7. 민감 데이터 처리 규칙

### 직원 개인정보
- 주민번호 뒷자리: 화면 표시 시 마스킹 처리 (`******`)
- 계좌번호: 뒤 4자리만 표시 (`****-**-***456`)
- 연락처: 중간 4자리 마스킹 (`010-****-5678`)

### 급여 및 지분 데이터
- 급여 정보: 권한 있는 화면에서만 조회 가능
- 지분 정산 내역: 대표님 확인 화면에서만 표시
- 로그 파일에 급여/지분 수치 출력 금지

### 세무 자료
- 세무사 제출용 리포트: Excel 파일로 출력 후 카톡/이메일 전달
- 파일 생성 후 다운로드 폴더 자동 열기
- 전달 이력 기록 (날짜, 파일명, 전달 방법)

## 8. 데이터 입력 유효성 검사

### 금액 입력
```
- 숫자만 입력 가능
- 음수 입력 불가 (별도 환불/취소 항목으로 처리)
- 최대값: 99,999,999원 (초과 시 경고)
- 천 단위 구분자 자동 표시
```

### 날짜 입력
```
- 미래 날짜 입력 시 경고 (일부 예외: 발주 예정일 등)
- 날짜 형식: YYYY-MM-DD 통일
- 달력 피커 기본 제공
```

### 필수 항목
```
- 빈 값 저장 시 빨간 테두리 + 안내 메시지
- 저장 버튼 클릭 전 유효성 자동 검사
- 에러 항목으로 자동 스크롤 이동
```

## 9. 데이터 내보내기 (Export)

### 지원 형식
```
Excel (.xlsx)  → 세무사 제출용 리포트, 급여 내역
CSV (.csv)     → POS 데이터 가져오기/내보내기
PDF (.pdf)     → 회의록, 계약서, 공식 문서 출력용
```

### 내보내기 항목별 용도
```
월별 매출/지출 내역  → 세무사 월간 자료 제출
급여 지급 내역       → 세무사 급여 자료 제출
동업자 정산 내역     → 내부 공유용
재고 현황            → 발주 계획 수립
```

## 10. 데이터 가져오기 (Import)

### POS 데이터 가져오기
```
지원 형식  : CSV, Excel
가져오기 전: 데이터 미리보기 표시
중복 확인  : 동일 날짜 데이터 존재 시 덮어쓰기 여부 확인
실패 처리  : 오류 행 목록 표시 후 수동 입력 안내
```

### 가져오기 공통 규칙
- 가져오기 전 반드시 백업 자동 생성
- 오류 발생 시 전체 롤백 (부분 저장 금지)
- 가져오기 결과 요약 표시 (성공 N건 / 실패 N건)
