# MonoDesk Git 버전 관리 정책 (Git Policy)

이 문서는 MonoDesk 프로젝트의 코드 변경 이력을 관리하는 기준입니다.
Git을 활용해 실수를 되돌리고, 변경 이력을 추적하며, 안전하게 개발을 진행합니다.

---

## 1. Git을 사용하는 이유 (비전문가용 설명)

Git은 쉽게 말해 **코드의 타임머신**이에요.

```
예시 상황:
어제까지 잘 되던 기능이 오늘 수정 후 갑자기 안 됨
→ Git이 있으면: "어제 버전으로 되돌려줘" 한 마디로 해결
→ Git이 없으면: 무엇이 바뀌었는지 찾기가 매우 어려움
```

MonoDesk는 실제 운영에 사용하는 프로그램이므로
**코드 변경 이력 관리는 필수**입니다.

---

## 2. 브랜치 전략

### 브랜치 구조
```
main
 └── 실제 사용 중인 안정 버전
 └── 직접 수정 금지 (반드시 develop에서 병합)

develop
 └── 개발 중인 최신 버전
 └── 새 기능은 여기서 feature 브랜치로 분기

feature/{기능명}
 └── 새 기능 개발 전용
 └── 완성 후 develop에 병합

fix/{버그명}
 └── 버그 수정 전용
 └── 완성 후 develop에 병합

design/{화면명}
 └── UI/UX 변경 전용
 └── 완성 후 develop에 병합

release/{버전}
 └── 배포 준비 브랜치
 └── 최종 테스트 후 main에 병합
```

### 브랜치 흐름 예시
```
새 기능 개발 흐름:

1. develop 브랜치에서 시작
   git checkout develop
   git pull

2. 새 브랜치 생성
   git checkout -b feature/세무-월별리포트

3. 기능 개발 및 커밋

4. develop에 병합
   git checkout develop
   git merge feature/세무-월별리포트

5. 테스트 완료 후 main에 병합
   git checkout main
   git merge develop

6. 사용 완료된 브랜치 삭제
   git branch -d feature/세무-월별리포트
```

## 3. 커밋 메시지 규칙

### 기본 형식
```
[타입] 변경 내용 요약 (50자 이내)

상세 설명 (필요한 경우, 72자 줄 바꿈)
- 변경한 이유
- 영향 받는 기능
- 주의사항
```

### 타입 종류
```
[기능추가]    새로운 기능 추가
[버그수정]    오류 수정
[디자인]      UI/UX 변경
[리팩토링]    기능 변경 없이 코드 구조 개선
[DB변경]      데이터베이스 구조 변경
[AI추가]      AI 기능 추가 또는 변경
[테스트]      테스트 코드 추가 또는 수정
[문서]        문서 수정 (코드 변경 없음)
[설정]        환경 설정, 패키지 변경
[보안]        보안 관련 수정
```

### 커밋 메시지 예시
```
✅ 좋은 예시
[기능추가] 세무/회계 월별 손익 계산 기능 추가
[버그수정] 매출 입력 시 날짜 저장 오류 수정
[디자인]   대시보드 KPI 카드 레이아웃 개선
[DB변경]   직원 테이블에 고용형태 컬럼 추가
[AI추가]   영수증 OCR Tesseract 연동 기능 추가
[기능추가] POS CSV 파일 가져오기 기능 추가

  - 토스포스 CSV 형식 지원
  - 중복 날짜 데이터 덮어쓰기 확인 기능 포함

❌ 나쁜 예시
수정함
버그 고침
작업중
test
```

### 커밋 단위 원칙
- 하나의 커밋 = 하나의 변경 목적
- 여러 기능을 한번에 커밋하지 않음
- 작업 단위가 완성될 때마다 커밋 (하루 1회 커밋 지양)

## 4. 중대한 변경 기준 및 처리

### 중대한 변경으로 분류되는 경우
```
- 새로운 모듈 추가 (세무/회계, 직원 관리 등)
- 데이터베이스 테이블 구조 변경
- AI 기능 추가 또는 변경
- 전체 레이아웃 대규모 변경
- 핵심 계산 로직 변경 (급여, 지분, 매출 분석)
- 외부 라이브러리 주요 버전 업그레이드
```

### 중대한 변경 처리 순서
```
1. 현재 상태 백업
   → 앱 내 [설정 → 백업 → 지금 백업] 실행

2. 새 브랜치 생성
   git checkout -b feature/{기능명}

3. 변경 작업 진행

4. 테스트 완료 확인
   → QA 에이전트 검토

5. history/ 폴더에 변경 이력 문서 작성

6. develop 브랜치에 병합

7. 충분한 테스트 후 main에 병합
```

## 5. 변경 이력 문서 (history/)

### 언제 작성하나요?
중대한 변경이 있을 때마다 `history/` 폴더에 문서를 추가합니다.

### 파일 명명 규칙
```
history/YYYYMMDD_변경내용요약.md

예시:
history/20240304_세무회계모듈추가.md
history/20240315_POS연동기능추가.md
history/20240401_DB직원테이블변경.md
```

### 이력 문서 양식
```markdown
# 변경 이력: {변경 내용 요약}

## 기본 정보
- 날짜: YYYY년 MM월 DD일
- 브랜치: feature/{기능명}
- 작업자: Claude Code

## 요청 내용
(대표님이 요청한 내용 요약)

## 변경한 내용
### 추가된 파일
- `경로/파일명.확장자` : 설명

### 수정된 파일
- `경로/파일명.확장자` : 변경 내용 설명

### 데이터베이스 변경
- 테이블명: 변경 내용

## 테스트 및 검증 방법
- [ ] 테스트 항목 1
- [ ] 테스트 항목 2
- [ ] 테스트 항목 3

## 주의사항
- 특이사항이나 주의할 점 기술

## 향후 작업 제안
- 다음에 추가하면 좋을 기능이나 개선 사항
```

## 7. 버전 관리 규칙

### 버전 번호 형식
```
v{메이저}.{마이너}.{패치}

메이저 (v1.0.0 → v2.0.0)
→ 전체 구조 변경, 대규모 기능 추가

마이너 (v1.0.0 → v1.1.0)
→ 새 모듈 추가, 주요 기능 추가

패치 (v1.0.0 → v1.0.1)
→ 버그 수정, 소소한 개선
```

### 버전별 태그 생성
```
각 안정 버전 배포 시 Git 태그 생성

git tag -a v1.0.0 -m "세무/회계 모듈 최초 배포"
git tag -a v1.1.0 -m "매출 분석 POS 연동 추가"
git tag -a v1.2.0 -m "영수증 OCR AI 기능 추가"
```

### 모듈별 예상 버전 로드맵
```
v0.1.0  → 프로젝트 초기 구조 설정
v0.2.0  → 세무/회계 모듈 완성
v0.3.0  → 대시보드 연결
v0.4.0  → 매출 분석 (POS 연동)
v0.5.0  → 재고/발주 관리
v0.6.0  → 메뉴 관리
v0.7.0  → 직원 관리
v0.8.0  → 법인 관리
v0.9.0  → 운영 관리
v1.0.0  → 전체 기본 기능 완성 (정식 배포)
v1.1.0  → 영수증 OCR AI 추가
v1.2.0  → 회의록 자동 정리 AI 추가
v1.3.0  → 매출 분석 AI 리포트 추가
v2.0.0  → 마케팅 자동화 모듈 (2단계 프로젝트)
```

## 8. 자주 쓰는 Git 명령어 (대표님용 참고)

```bash
# 현재 상태 확인 (뭐가 바뀌었는지)
git status

# 변경 내용 저장 (커밋)
git add .
git commit -m "[기능추가] 세무 월별 리포트 추가"

# 브랜치 목록 확인
git branch

# 새 브랜치 만들고 이동
git checkout -b feature/새기능명

# 브랜치 이동
git checkout develop

# 이전 버전으로 되돌리기 (마지막 커밋 취소)
git revert HEAD

# 특정 버전으로 되돌리기
git checkout v1.0.0

# 변경 이력 보기
git log --oneline
```

## 9. 비상 상황 대처

### 기능이 갑자기 안 될 때
```
1. 당황하지 않기 😊
2. git log --oneline 으로 최근 커밋 확인
3. 문제 발생 직전 커밋으로 되돌리기
   git revert {커밋ID}
4. 저에게 상황 설명하면 같이 해결!
```

### 실수로 파일 삭제했을 때
```
git checkout -- {파일경로}
→ 마지막 커밋 상태로 파일 복구
```

### 완전히 처음부터 다시 하고 싶을 때
```
→ 백업 파일에서 복원
→ 앱 내 [설정 → 백업 → 복원] 사용
```
