# MonoDesk AI 사용 정책 (AI Policy)

이 문서는 MonoDesk에서 AI 기능을 개발하고 사용할 때 따르는 기준입니다.
모든 AI 처리는 로컬에서만 이루어지며, 비즈니스 데이터는 외부로 절대 유출되지 않습니다.

---

## 1. 핵심 원칙

- **로컬 전용**: 모든 AI 처리는 반드시 사용자 PC 내에서만 실행
- **데이터 보호**: 비즈니스 데이터(매출, 직원, 지분 등)의 외부 전송 절대 금지
- **사람이 최종 결정**: AI 처리 결과는 항상 사용자 확인 후 저장 (자동 저장 금지)
- **대체 수단 보장**: AI 기능 실패 시 수동 입력으로 반드시 대체 가능
- **투명성**: AI가 어떤 근거로 결과를 냈는지 사용자에게 설명 표시

---

## 2. AI 기술 스택

### 핵심 도구
```
LLM (텍스트 생성/분석)
└── Ollama
    ├── llama3.2     → 기본 텍스트 분석, 요약, 리포트 생성 (한국어 우수)
    └── mistral      → 빠른 응답이 필요한 간단한 작업

비전 (이미지 인식)
└── Ollama llava     → 수기 영수증 인식, 문서 이미지 분석

음성 인식
└── faster-whisper   → 회의 음성 → 텍스트 변환 (로컬 실행)
    ├── medium 모델  → 기본 사용 (한국어 인식률 우수)
    └── large 모델   → 정확도 필요 시 사용 (속도 느림)

OCR (문자 인식)
└── Tesseract 5.x    → 인쇄체 영수증, 문서 텍스트 추출
    └── kor+eng 언어팩 설치 필수

벡터 데이터베이스 (문서 검색)
└── ChromaDB         → 회의록, 계약서, 문서 자연어 검색

AI 파이프라인 연결
└── LangChain        → Ollama ↔ FastAPI 연결 브릿지
└── DSPy             → 복잡한 AI 워크플로우 (Phase 3에서 도입)
```

### 도구별 설치 위치
```
Ollama        → https://ollama.com (Windows 설치)
faster-whisper → pip install faster-whisper
Tesseract     → https://github.com/UB-Mannheim/tesseract/wiki (Windows)
ChromaDB      → pip install chromadb
LangChain     → pip install langchain langchain-community
```

## 3. AI 기능별 상세 정책

### 3-1. 영수증 OCR (이미지 → 지출 데이터)

**처리 흐름**
```
영수증 이미지 업로드
    │
    ▼
이미지 분류 (인쇄체 / 수기 자동 판단)
    │
    ├─ 인쇄체 → Tesseract OCR 처리
    └─ 수기   → Ollama llava 처리
    │
    ▼
추출 데이터 미리보기 표시
(날짜 / 거래처 / 품목 / 금액 / 분류)
    │
    ▼
사용자 확인 및 수정
    │
    ▼
최종 저장 (원본 이미지 함께 보관)
```

**추출 항목**
```
- 날짜 (YYYY-MM-DD 형식으로 변환)
- 거래처명
- 품목 (여러 개일 경우 리스트로 표시)
- 금액 (부가세 포함/별도 구분)
- 결제 수단 (현금/카드)
```

**저장 규칙**
- 원본 이미지: `/uploads/receipts/YYYY/MM/` 에 보관
- 파일명: `receipt_YYYYMMDD_거래처명.jpg`
- AI 인식률 낮을 시 (60% 미만): 수동 입력 안내 표시
- 원본 이미지는 5년간 보관 (세법 기준)

**오류 처리**
- Ollama 미실행: "AI 서버가 실행되지 않았습니다. 수동으로 입력해주세요." 안내
- 인식 실패: "인식에 실패했습니다. 이미지를 확인하거나 직접 입력해주세요." 안내

### 3-2. 회의록 자동 정리 (음성 → 문서)

**처리 흐름**
```
음성 녹음 파일 업로드 (mp3/wav/m4a)
    │
    ▼
faster-whisper → 텍스트 변환
(한국어 자동 감지)
    │
    ▼
Ollama llama3.2 → 회의록 자동 정리
(참석자 / 안건 / 결정사항 / 향후 과제)
    │
    ▼
양식에 맞춰 초안 생성
    │
    ▼
사용자 검토 및 수정
    │
    ▼
최종 저장 (원본 음성 + 완성 회의록 함께 보관)
```

**자동 생성 회의록 양식**
```
━━━━━━━━━━━━━━━━━━━━━━━━
[회의명]
일시    : YYYY년 MM월 DD일 HH:MM
장소    :
참석자  :
작성자  : (AI 초안 / 최종 확인자: )
━━━━━━━━━━━━━━━━━━━━━━━━

1. 안건
   -

2. 논의 내용
   -

3. 결정 사항
   -

4. 향후 과제 및 담당자
   - 과제:  / 담당:  / 기한:

5. 기타
   -
━━━━━━━━━━━━━━━━━━━━━━━━
```

**저장 규칙**
- 원본 음성: `/uploads/audio/YYYY/MM/`
- 회의록 문서: `/uploads/documents/minutes/`
- 파일명: `minutes_YYYYMMDD_회의명`
- ChromaDB에 벡터 인덱싱 (검색 가능하도록)

### 3-3. 매출 데이터 AI 분석

**처리 흐름**
```
POS 데이터 + 지출 데이터 로드
    │
    ▼
Ollama llama3.2 분석
    │
    ▼
자연어 리포트 생성
    │
    ▼
대시보드에 인사이트 표시
```

**제공 분석 항목**
```
- 이번 달 매출 요약 및 전월 비교
- 인기 메뉴 TOP 5 / 비인기 메뉴 분석
- 요일별/시간대별 매출 패턴
- 원가율 분석 및 마진 개선 제안
- 재고 소진 예측 및 발주 추천
- 계절성 분석 (제철 해산물 특성 반영)
```

**리포트 표시 형식**
```
┌─────────────────────────────────┐
│  ✨ AI 경영 인사이트             │
│  분석 기준: 2024년 3월          │
├─────────────────────────────────┤
│  📈 이번 달 매출은 지난달 대비  │
│  8% 증가했습니다.               │
│                                  │
│  🍣 활전복이 가장 높은 매출     │
│  기여도(23%)를 보였습니다.      │
│                                  │
│  ⚠️  화요일 매출이 평균 대비    │
│  40% 낮습니다. 화요일 특별      │
│  프로모션을 고려해보세요.        │
└─────────────────────────────────┘
```

### 3-4. 문서 자연어 검색

**처리 흐름**
```
사용자 자연어 검색 입력
(예: "작년 임대차 계약 만료일")
    │
    ▼
ChromaDB 벡터 검색
    │
    ▼
관련 문서 TOP 3 반환
    │
    ▼
결과 + 출처 문서 표시
```

**검색 가능 문서 범위**
```
- 주주총회 회의록
- 계약서 (임대차, 거래처)
- 법인 정관
- 운영 일지
- 발주 내역
```

**검색 결과 표시 형식**
```
검색어: "임대차 계약 만료"
─────────────────────────────
📄 임대차계약서_20230301.pdf
   "...계약 만료일은 2025년 3월 1일이며..."
   → [문서 열기]

📄 법인관리_회의록_20231201.md
   "...임대차 계약 갱신 논의..."
   → [문서 열기]
```

## 4. Ollama 모델 관리

### 설치 권장 모델
```
ollama pull llama3.2      # 기본 텍스트 분석 (한국어 우수)
ollama pull llava         # 이미지/영수증 인식
ollama pull mistral       # 빠른 응답용 보조 모델
```

### 모델 설정값
```python
# 기본 설정
OLLAMA_BASE_URL = "http://localhost:11434"
DEFAULT_MODEL = "llama3.2"
VISION_MODEL = "llava"
MAX_TOKENS = 2048
TEMPERATURE = 0.3        # 낮을수록 일관된 답변 (비즈니스 분석용)
LANGUAGE = "Korean"      # 응답 언어 한국어 고정
```

### Ollama 상태 확인
- 앱 시작 시 Ollama 실행 여부 자동 확인
- 미실행 시 상단 배너로 안내
- AI 기능 버튼 비활성화 (기본 기능은 정상 동작)

## 5. 성능 및 하드웨어 권장 사양

### 최소 사양
```
CPU  : 4코어 이상
RAM  : 8GB 이상
저장 : SSD 20GB 이상 여유 공간 (모델 파일 포함)
```

### 권장 사양
```
CPU  : 8코어 이상
RAM  : 16GB 이상
GPU  : NVIDIA GPU (VRAM 6GB 이상) → 처리 속도 대폭 향상
저장 : SSD 50GB 이상 여유 공간
```

### 모델별 처리 속도 (CPU 기준 참고)
```
llama3.2 텍스트 분석  : 약 10~30초
llava 이미지 인식     : 약 20~60초
faster-whisper 음성   : 실시간의 약 0.5~2배
```

## 6. 보안 원칙

- 모든 AI 처리는 로컬(localhost)에서만 실행
- Ollama API는 외부 접근 차단 (127.0.0.1만 허용)
- AI 처리 중 민감 데이터(급여, 지분) 로그 출력 금지
- ChromaDB 벡터 데이터 로컬 저장 (`/database/chromadb/`)
- AI 기능 관련 외부 API 키 사용 금지

## 7. 단계별 개발 계획

### Phase 1 — 기본 기능 완성 (AI 없이)
```
세무/회계, 매출 분석, 재고, 메뉴, 직원, 법인 관리
→ AI 없이도 모든 기본 기능 100% 동작
```

### Phase 2 — AI 기능 추가
```
① 영수증 OCR (Tesseract + llava)
② 회의록 자동 정리 (Whisper + llama3.2)
③ 기본 매출 분석 리포트 (llama3.2)
```

### Phase 3 — AI 고도화
```
④ ChromaDB 문서 검색
⑤ DSPy 기반 복잡한 분석 파이프라인
⑥ 재고 소진 예측 및 발주 자동 추천
⑦ 계절성 메뉴 전략 AI 제안
```
