# 대시보드 일별/주별/월별 뷰 전환 기능 구현

## 날짜
2026-04-07

## 요청 내용
기존 월별 단일 대시보드에 일별/주별/월별 뷰 전환 기능을 추가한다.
뷰 토글(탭) 방식으로 전환하며, 각 뷰는 독립 컴포넌트로 구성한다.

## 변경 파일 목록

### 백엔드
- `backend/schemas/dashboard.py` — 새 응답 스키마 5개 추가
  - DailyKpiResponse, WeeklyKpiResponse, WeeklyDailyBreakdown
  - MonthlyKpiResponse, MonthlyWeeklyTrend, MonthlyDailyTrend
- `backend/services/dashboard_service.py` — 새 서비스 함수 추가
  - `_get_day_sales()`: 특정 날짜 매출 집계 (이중 계산 방지 적용)
  - `_get_day_expense()`: 특정 날짜 지출 집계
  - `_count_business_days()`: 월별 영업일 수 계산
  - `get_daily_kpi()`: 일별 KPI 집계
  - `get_weekly_kpi()`: 주별 KPI 집계 (월~일)
  - `get_monthly_kpi()`: 월별 상세 KPI 집계
- `backend/routers/dashboard.py` — API 엔드포인트 3개 추가
  - `GET /api/dashboard/daily?date=YYYY-MM-DD`
  - `GET /api/dashboard/weekly?date=YYYY-MM-DD`
  - `GET /api/dashboard/monthly?year=&month=`

### 프론트엔드
- `frontend/src/api/dashboardApi.js` — API 함수 3개 추가
  - `getDailyKpi(date)`, `getWeeklyKpi(date)`, `getMonthlyKpi(year, month)`
- `frontend/src/pages/DashboardPage.jsx` — 뷰 토글 방식으로 재구성
- `frontend/src/components/modules/dashboard/DailyDashboard.jsx` — 일별 뷰 메인
- `frontend/src/components/modules/dashboard/DailyKpiRow.jsx` — 일별 KPI 카드 2행
- `frontend/src/components/modules/dashboard/DailyCompareBanner.jsx` — 전일/전주 비교 배너
- `frontend/src/components/modules/dashboard/WeeklyDashboard.jsx` — 주별 뷰 메인
- `frontend/src/components/modules/dashboard/MonthlyDashboard.jsx` — 월별 뷰 메인
- `frontend/src/components/modules/dashboard/MonthlyCalendarHeatmap.jsx` — 달력 히트맵

## 주요 구현 사항

### 일별 뷰 (DailyDashboard)
- 날짜 네비게이션 (이전/다음/오늘 버튼, 오늘 이후 불가)
- KPI 카드 8개 (2행): 당일 매출/달성률/고객수/단가 + 지출/순이익/할인/서비스
- 전일 대비 금액 차이, 전주 동요일 대비 증감률 배너
- 월 누적 현황 (누적 매출/지출/순이익, 영업일 진행 바)
- 결제수단별 도넛 차트 (카드/현금/배달)

### 주별 뷰 (WeeklyDashboard)
- 주 네비게이션 (월~일 기준, n월 n주차 표시)
- KPI 카드 4개: 주간 매출/지출/순이익/고객수
- 요일별 매출 막대 차트 (최고일: 파란색, 휴무: 회색)
- 베스트/워스트 요일 배지, 전주 대비 요약

### 월별 뷰 (MonthlyDashboard)
- 월 네비게이션 (이번 달 이후 불가)
- KPI 카드 4개: 월 매출/순이익/이익률/일평균 매출
- 손익 구조 트리 (매출 → 원재료비/인건비/고정비/순이익)
- 달력 히트맵 (매출 높을수록 진한 파란색)
- 주차별 매출/지출 바 차트 (1~4주차)

## 비즈니스 로직 특이사항
- 이중 계산 방지: SalesRecord(is_pos_synced==0) + PosSalesRaw 합산 방식 유지
- 영업일 수: 실제 매출 기록이 있는 날 기준, 최소 25일 보장
- 목표 매출: 전월 매출 / 영업일 수 (전월 데이터 없으면 0)
- 원재료비 분류: 카테고리명에 "원재료", "식자재", "주류" 포함 건
- 고정비 분류: 카테고리명에 "고정", "임대", "관리비" 포함 건

## 테스트 방법
1. 백엔드 서버 실행: `cd backend && uvicorn main:app --reload`
2. 프론트엔드 실행: `cd frontend && npm run dev`
3. 대시보드 페이지 접속 → 상단 [일별|주별|월별] 탭 클릭하여 뷰 전환
4. 일별: 날짜 네비게이션으로 날짜 변경, 오늘 이후 버튼 비활성화 확인
5. 주별: 이전/다음 주 이동, 요일별 막대그래프, 베스트/워스트 배지 확인
6. 월별: 달력 히트맵 색상 표시, 손익 구조 트리, 주차별 차트 확인
7. API 직접 테스트:
   - GET http://localhost:8000/api/dashboard/daily?date=2026-04-07
   - GET http://localhost:8000/api/dashboard/weekly?date=2026-04-07
   - GET http://localhost:8000/api/dashboard/monthly?year=2026&month=4
