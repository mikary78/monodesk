## DB 구조 (신규 테이블 5개)

```
inventory_categories     — 재고 분류 (수산물, 채소, 주류 등)

purchase_order_items     — 발주 품목 (발주수량, 단가, 실입고수량)
```

## 핵심 비즈니스 로직

- **재고 수량 자동 관리**: 수량 조정(입고/출고/폐기) 시 InventoryItem.current_quantity 자동 갱신
- **음수 재고 방지**: 출고·폐기 시 현재 수량보다 많은 수량 조정 불가
- **발주 번호 자동 채번**: `PO-YYYYMMDD-NNN` 형식으로 날짜별 일련번호 부여
- **입고 처리 자동화**: 입고 완료 시 각 품목 재고 자동 증가 + 조정 이력 자동 생성
- **재고 부족 알림**: current_quantity <= min_quantity 인 품목을 실시간으로 배너 표시
- **소프트 삭제 패턴**: is_deleted=1 로 논리 삭제, 이력 보존

## 제철해산물 주점 특화 기본 분류 (seed-categories)

- 수산물 (수산물·활어·패류)
- 채소/과일 (신선 채소·과일·허브)
- 주류 (소주·맥주·막걸리·와인)
- 음료/기타식재료 (음료·조미료·건식품)
- 포장재/용기 (포장재·일회용기)
- 소모품 (청소용품·위생용품·주방소모품)

## 테스트 및 검증 방법

1. 백엔드 서버 실행: `uvicorn main:app --reload --port 8000`
2. Swagger UI 접속: `http://localhost:8000/docs`
3. `/api/inventory/seed-categories` POST → 기본 분류 생성
4. 품목 등록 → 수량 조정(입고) → 재고 반영 확인
5. 발주서 생성 → 입고 처리 → 재고 자동 증가 확인
6. 재고 부족 임계값 이하로 수량 조정 → 배너 알림 표시 확인

## 향후 작업 제안 및 주의사항

- **Step 4 — 메뉴 관리**: 재고 품목과 메뉴를 연동하여 메뉴 판매 시 자동 재고 차감 가능
- **바코드 스캔 연동**: 향후 USB 바코드 리더기로 입고 처리 자동화 가능
- **이동 평균 단가**: 입고 단가가 바뀔 때 평균 단가 계산 기능 추가 검토 필요
- **월별 원가 분석**: inventory_adjustments 데이터를 accounting 모듈과 연계하여 식재료비 자동 집계 가능
