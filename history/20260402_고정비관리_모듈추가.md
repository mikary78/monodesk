# 변경 이력: 고정비 관리 모듈 추가

## 기본 정보
- 날짜: 2026년 04월 02일
- 브랜치: master 직접 작업 (정책 위반 — 이후 feature 브랜치 사용 예정)
- 작업자: Claude Code

## 요청 내용
엑셀 "0.기초및고정비" 시트의 고정비 관리 기능을 MonoDesk에 구현.
월초 1회 설정 → 매월 자동 복사, 설정금액(예산) vs 실제금액 비교 구조.
손익계산 API에 고정비 실제지출 합산 연동 포함.

## 변경한 내용

### 추가된 파일
- `backend/migrations/011_fixed_costs.sql` : fixed_cost_items, fixed_cost_records 테이블 생성
- `frontend/src/components/modules/operations/FixedCostSettings.jsx` : 마스터 항목 관리 화면
- `frontend/src/components/modules/operations/FixedCostMonthly.jsx` : 월별 실적 비교 화면

### 수정된 파일
- `backend/models/operations.py` : FixedCostItem, FixedCostRecord 모델 추가
- `backend/schemas/operations.py` : 6개 스키마 추가 (Item/Record Create/Update/Response, Monthly/CategoryTotal)
- `backend/schemas/accounting.py` : ProfitLossResponse에 total_fixed_cost, total_combined_expense 필드 추가
- `backend/services/operations_service.py` : 마스터 CRUD + 월별 자동생성 + 요약 + 손익연동 함수 추가
- `backend/services/accounting_service.py` : calculate_profit_loss()에 고정비 합산 연동
- `backend/routers/operations.py` : 고정비 7개 엔드포인트 추가
- `frontend/src/pages/OperationsPage.jsx` : "고정비 설정", "월별 고정비" 탭 추가 (9개 탭)
- `frontend/src/api/operationsApi.js` : 고정비 7개 API 클라이언트 함수 추가

### 데이터베이스 변경
- `fixed_cost_items` 테이블 신규 생성
  - 기초 데이터 14개 자동 삽입 (facility 5개, operation 9개)
  - facility: 임대료, 도시가스, 전기료, 수도료, 건물관리비
  - operation: 화재보험, 보안설비, 방역, 정수기, 인터넷/전화, POS, 테이블오더, SMS수수료, 세무기장료
- `fixed_cost_records` 테이블 신규 생성
  - UNIQUE(item_id, year, month) — 월별 1개 레코드 보장

### 핵심 비즈니스 로직
- **월초 자동 세팅**: GET 요청 시 해당 월 레코드 없으면 마스터 default_amount 복사 자동 생성
- **손익 연동**: `total_expense(변동비) + total_fixed_cost(고정비) = total_combined_expense`
- **차이 계산**: variance = actual - default (양수=초과, 음수=절감)

## 테스트 및 검증 방법
- [ ] 운영관리 → 고정비 설정 탭: 14개 항목 두 섹션(시설비/운영)으로 표시
- [ ] 항목 추가 모달 → 카테고리 선택 후 저장
- [ ] 인라인 수정 → 업체명/이체일/설정금액 변경 후 체크 저장
- [ ] 비활성화 → 목록에서 사라지고 이후 월별 기록 미생성
- [ ] 월별 고정비 탭: 현재 월 자동 로드, 14개 행 표시
- [ ] 요약 카드 3개 (설정/실제/차이) 정확한 합계 표시
- [ ] 행 클릭 → 실제금액/납부일/메모 인라인 편집 저장
- [ ] 세무/회계 → 손익계산에서 total_fixed_cost 필드 확인

## 주의사항
- 마스터 항목 비활성화 후에는 기존 월별 레코드는 유지되나 신규 생성 안 됨
- 손익계산 API 응답 스키마 변경으로 프론트엔드 ProfitLoss 컴포넌트 업데이트 권장
