# 직원 관리 모듈 (Step 5) 개발

## 날짜
2026-03-09

## 요청 내용 요약
MonoDesk 5번째 모듈인 직원 관리 기능 전체 구현 요청.
직원 CRUD, 출퇴근 기록, 급여 계산(한국 노동법 기준), 근로계약서 파일 관리, 급여 이력 조회까지 백엔드·프론트엔드 완전 구현.

## 변경한 내용

### 신규 파일 (백엔드)
- `backend/models/employee.py` — SQLAlchemy 모델 3개 정의
  - `Employee`: 직원 기본 정보 (고용형태, 급여유형, 4대보험, 계약서 경로 등)
  - `AttendanceRecord`: 출퇴근 기록 (출근/퇴근 시각, 실근무시간, 연장/야간 시간)
  - `SalaryRecord`: 월별 급여 정산 (기본급, 주휴수당, 연장/야간수당, 4대보험 공제, 실수령액)
- `backend/schemas/employee.py` — Pydantic V2 스키마
  - Create/Read/Update 스키마, SalaryCalculationResult, MonthlySalarySummary 포함
- `backend/services/employee_service.py` — 비즈니스 로직
  - 2026년 최저임금 10,030원 적용
  - 주휴수당 계산 (근로기준법 제55조, 주 15시간 이상 근무 시)
  - 연장근로수당 1.5배, 야간근로수당 0.5배 가산 (22:00~06:00)
  - 휴게시간 자동 차감 (8시간 이상 근무 시 1시간 공제)
  - 4대보험 공제 (국민연금 4.5%, 건강보험 3.545%, 장기요양 12.95%, 고용보험 0.9%)
  - 급여 계산 미리보기 및 DB 저장 (upsert 방식)
  - 근로계약서 파일 경로 업데이트 함수
  - 직원 전체 급여 이력 조회 함수
- `backend/routers/employee.py` — FastAPI 라우터
  - 직원 CRUD: GET/POST /employees, GET/PUT/DELETE /employees/{id}
  - 근로계약서: POST/GET/DELETE /employees/{id}/contract
  - 출퇴근: GET /attendance, GET /attendance/calculate-hours, POST /attendance, PUT/DELETE /attendance/{id}
  - 급여: GET /salary/overview, POST /salary/calculate, POST /salary/save, GET /salary, PUT /salary/{id}, GET /salary/history
  - 파일 저장 경로: `backend/uploads/contracts/{employee_id}/`
  - 허용 파일 형식: PDF, JPG, JPEG, PNG (10MB 제한)

## 주요 기술 결정 사항

### FastAPI 라우트 순서 문제
정적 경로(`/salary/overview`, `/attendance/calculate-hours`)를 동적 경로(`/salary/{id}`, `/attendance/{id}`) 앞에 등록해야
FastAPI가 문자열을 정수 ID로 잘못 파싱하는 422 오류 방지 가능.

### 근로계약서 업로드 아키텍처
React `useRef`로 숨겨진 `<input type="file">` 관리,
`contractTargetId` state로 업로드 대상 직원 추적,
`FormData` 사용 시 `Content-Type` 헤더 미설정 (브라우저 자동 처리).

### 주휴수당 월 환산
월평균 주 수 4.33 근사치로 주평균 근무시간 산출,
`(주근무시간 / 40) × 8 × 시급 × 4` 공식 적용.

## 향후 작업 제안 및 주의사항

1. **은행 계좌 암호화**: 현재 평문 저장 중. 추후 SQLCipher 또는 앱 레벨 암호화 적용 권장.
2. **파일 저장소 이전**: `backend/uploads/`는 재배포 시 초기화 위험. 영구 저장 필요 시 OS 수준 별도 경로 지정 고려.
3. **근로계약서 OCR**: Tesseract 연동으로 계약서 내용 자동 파싱 가능 (AI 모듈 Phase 2).
4. **대시보드 연동**: 직원 수, 이달 인건비 KPI를 대시보드에 노출 예정 (Step 6).
5. **4대보험 요율 변경**: 정부 고시 변경 시 `employee_service.py` 상수값 수동 업데이트 필요.
