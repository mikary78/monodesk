# 누락 탭 컴포넌트 연결 및 __pycache__ 추적 제거

## 변경 요약
- 이전에 탭에 연결되지 않은 3개 컴포넌트를 신규 작성 후 페이지에 연결
- inventoryApi.js 누락 함수 8개 추가
- backend/__pycache__ 디렉터리 전체를 git 추적에서 제거

## 배경
기존에 탭 구조는 정의되어 있었으나 실제 컴포넌트 파일이 저장되지 않아 탭 클릭 시 오류 발생.
또한 Python 컴파일 캐시(.pyc) 파일이 .gitignore에 선언되어 있었음에도 이미 커밋된 상태라 계속 git status에 나타나는 문제를 `git rm -r --cached`로 해결.

## 신규 구현 컴포넌트

### AttendanceCalendar.jsx
- 직원×날짜 그리드 (가로: 날짜, 세로: 직원)
- 셀 클릭 → 9가지 근태 상태 드롭다운 선택
- API: GET /api/employee/attendance/monthly-calendar
- API: PATCH /api/employee/attendance/{id}/status

### DailyPriceGrid.jsx
- 날짜×품목 가로 스크롤 그리드
- 셀 클릭 → 팝오버 입력 (수량/단가)
- 전일 대비 가격 변동 색상 표시 (상승=빨강, 하락=파랑)
- API: GET /api/inventory/daily-price, PATCH /api/inventory/daily-price

### InventorySnapshot.jsx
- snapshotType prop으로 월초/월말 구분 (snapshot-start / snapshot-end)
- 카테고리별 테이블 (품목명, 단위, 수량, 단가, 금액)
- 확정 후 잠금 처리 (수정 불가)
- 월말 확정 시 다음달 월초 자동 이월 안내
- API: GET /api/inventory/snapshot, POST /api/inventory/snapshot/confirm

## 변경 파일
- `frontend/src/components/modules/employee/AttendanceCalendar.jsx` (신규)
- `frontend/src/components/modules/inventory/DailyPriceGrid.jsx` (신규)
- `frontend/src/components/modules/inventory/InventorySnapshot.jsx` (신규)
- `frontend/src/pages/EmployeePage.jsx` (근무표 달력 탭 연결)
- `frontend/src/pages/InventoryPage.jsx` (데일리단가/월초재고/월말재고 탭 추가 → 총 6탭)
- `frontend/src/api/inventoryApi.js` (누락 함수 8개 추가)

## InventoryPage 탭 구성 변경
| 탭 ID | 탭명 | 컴포넌트 |
|-------|------|---------|
| items | 재고 현황 | InventoryList |
| orders | 발주 관리 | PurchaseOrder |
| history | 입출고 내역 | InventoryHistory |
| daily-price | 데일리 단가 | DailyPriceGrid ← 신규 연결 |
| snapshot-start | 월초 재고 | InventorySnapshot(start) ← 신규 연결 |
| snapshot-end | 월말 재고 | InventorySnapshot(end) ← 신규 연결 |

## __pycache__ 추적 제거
- `git rm -r --cached backend/**/__pycache__/` 로 43개 .pyc 파일 추적 해제
- .gitignore에 `__pycache__/`, `*.pyc` 이미 선언되어 있었으나, 이전 커밋에 포함된 상태라 무시되지 않음
- 추적 해제 후 이후 빌드 생성되는 .pyc 파일은 자동 무시됨
