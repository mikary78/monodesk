# 매입 출처 구분 모듈 추가 — 엑셀 3.원·부재료 시트 구현

## 변경 요약
- `inventory_adjustments` 테이블에 `purchase_source`, `linked_expense_id` 컬럼 추가
- 본사구매 / 현장구매(법카) / 현장구매(시재) / 기타 4가지 출처 구분
- 월별 매입 출처별 집계 API 추가 (`GET /api/inventory/purchases/summary/{year}/{month}`)
- 입고 처리 폼에 매입 출처 선택 UI 추가 (발주서 단위로 선택)
- 대시보드 KPI 응답에 매입 출처별 합계 연동

## 배경
엑셀 3.원·부재료 시트에서 본사구매(계좌이체) / 현장구매(법카) / 현장구매(시재)를
구분하여 ★보고서의 원재료 지출 집계와 자동 연동하기 위해 구현.
기존에는 입고 처리 시 구매 경로 구분이 없어 집계 보고서를 수동으로 작성해야 했음.

## 매입 출처 코드 정의
| 코드 | 한국어 | 설명 |
|------|--------|------|
| `headquarters` | 본사구매 (계좌이체) | 법인 계좌에서 이체로 결제 |
| `site_card` | 현장구매 법카 | 현장에서 법인카드로 결제 |
| `site_cash` | 현장구매 시재 | 현장에서 현금(시재)으로 결제 |
| `direct` | 기타 직접구매 | 위 분류 외 구매 (기본값) |

- `frontend/src/components/modules/inventory/PurchaseOrderTab.jsx` — `ReceiveOrderModal`에 매입 출처 선택 UI 추가
- `frontend/src/api/inventoryApi.js` — `getPurchaseSummary` 함수 추가

## 집계 기준
- 대상: `adjustment_type = '입고'` 기록만
- 금액: `|quantity_change| × unit_price`
- 전체 4가지 출처를 항상 포함 (데이터 없으면 금액 0, 건수 0으로 표시)
- 기존 입고 기록의 `purchase_source`는 NULL → 'direct'로 처리

## DB 마이그레이션
```sql
ALTER TABLE inventory_adjustments ADD COLUMN purchase_source TEXT DEFAULT 'direct';
ALTER TABLE inventory_adjustments ADD COLUMN linked_expense_id INTEGER;
CREATE INDEX IF NOT EXISTS idx_adj_purchase_source ON inventory_adjustments(purchase_source);
```
