## 구현된 API 엔드포인트
- `POST /api/sales-analysis/import` — CSV 파일 업로드
- `GET /api/sales-analysis/imports` — 가져오기 이력
- `DELETE /api/sales-analysis/imports/{id}` — 이력 삭제
- `GET /api/sales-analysis/summary` — 월별 KPI 요약
- `GET /api/sales-analysis/trend` — 매출 트렌드 (daily/weekly/monthly)
- `GET /api/sales-analysis/menu-analysis` — 메뉴 분석
- `GET /api/sales-analysis/time-analysis` — 시간대/요일 분석
- `GET /api/sales-analysis/payment-analysis` — 결제 수단 분석
- `GET /api/sales-analysis/menus` — 메뉴 마스터
- `PUT /api/sales-analysis/menus/{id}` — 메뉴 수정
- `POST /api/sales-analysis/targets` — 목표 매출 등록/수정
- `GET /api/sales-analysis/targets/{year}/{month}` — 목표 조회
- `POST /api/sales-analysis/ai-insight` — AI 인사이트 생성

## DB 테이블 신규 생성
- `pos_imports` — POS 파일 가져오기 이력
- `pos_sales_raw` — POS 원본 판매 데이터
- `menu_items` — 메뉴 마스터
- `sales_targets` — 월별 목표 매출

## 테스트 및 검증 방법
1. 백엔드 실행: `uvicorn main:app --reload --port 8000`
2. API 문서 확인: `http://localhost:8000/docs`
3. 프론트엔드 실행: `npm run dev` (포트 5173)
4. 사이드바 "매출 분석" 클릭 → 6개 탭 확인
5. CSV 샘플 파일 업로드 → 데이터 파싱 확인
6. 트렌드, 메뉴, 시간대, 결제 탭 차트 렌더링 확인

## 주의사항
- POS CSV 컬럼명은 자동 감지 (한국어/영어 혼용 지원)
- Ollama 미실행 시 AI 인사이트는 기본 텍스트 분석으로 대체
- 동일 파일 재업로드 시 중복 감지 (SHA256 해시 기반)
- 취소된 주문은 매출 집계에서 자동 제외
