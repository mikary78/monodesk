# 근무표 주별 뷰 전환 + 일급 직원 추가

**날짜**: 2026-04-20  
**브랜치**: feature/attendance-weekly-view  
**작업자**: Claude Sonnet 4.6

---

## 작업 요약

### 1. 근무표 주별 뷰 전환 (AttendanceCalendar.jsx 전면 재작성)
- 기존 월별 달력 뷰 → 주별 뷰로 전면 재작성
- 상단 네비게이션: 이전 주 / 다음 주 / 이번 주 버튼
- 헤더 표시: `2026년 4월 3주차 (4/14~4/20)` 형식
- 계약형태별 섹션 구분 (4대보험 / 3.3% / 시급알바 / 일급)
- 셀 클릭 → 상태 선택 드롭다운 (즉시 bulk API 저장)
- 우클릭(onContextMenu) → 메모 입력 팝오버
- 오늘 날짜 강조 표시 (파란색 배경)
- 우측 합계: `N / 7일` 형식 근무일수
- 하단 행: 날짜별 근무 인원수

### 2. 새 백엔드 API 추가 (backend/routers/employee.py)
- `GET /api/employee/attendance/weekly?date=YYYY-MM-DD` 추가
  - 기준 날짜가 포함된 주(월~일) 7일 데이터 반환
  - 기록 없는 날짜는 `{ "status": null, "memo": null }` 반환
  - 주차 번호 포함 (`week_number`)
- `PATCH /api/employee/attendance/bulk` 추가
  - 여러 직원 근태 기록 일괄 UPSERT
  - 기존 레코드: UPDATE / 없으면: INSERT
  - 두 엔드포인트 모두 `require_role("admin", "manager", "staff")` 적용

### 3. 비즈니스 로직 추가 (backend/services/employee_service.py)
- `get_weekly_calendar(db, date_str)` — 주별 달력 데이터 반환
- `bulk_update_attendance(db, records)` — 근태 일괄 UPSERT
- `calculate_daily_wage_monthly(db, employee_id, year, month)` — 일급 직원 월 급여 계산 (일급 × 근무일수)

### 4. 일급 계약형태 추가
- `ContractType` Enum에 `DAILY = "일급"` 추가 (models/employee.py)
- `Employee` 모델에 `daily_wage` 컬럼 추가 (Integer, default=0)
- 스키마 업데이트: EmployeeBase, EmployeeUpdate, EmployeeResponse에 `daily_wage` 필드 추가
- DB 마이그레이션 파일 생성: `018_daily_wage.sql`

### 5. 직원 등록 폼 업데이트 (EmployeeFormModal.jsx)
- 계약형태 선택지에 '일급' 추가
- 일급 선택 시 → 일급(원/일) 입력 필드 표시
- `daily_wage` 필드 유효성 검사 및 API 연동

### 6. API 함수 추가 (frontend/src/api/employeeApi.js)
- `getWeeklyAttendance(date)` 추가
- `bulkUpdateAttendance(records)` 추가
- `formatContractType()` 맵에 `"일급"` 추가

## 참고 사항
- 기존 `GET /attendance/monthly-calendar` 엔드포인트는 삭제하지 않고 유지
- `AttendanceRecord.work_date`, `AttendanceRecord.daily_status`, `AttendanceRecord.memo` 컬럼 사용
- 주(week)는 월요일 시작 기준 (ISO 8601 준수)
- Optimistic UI 적용: 셀 변경 즉시 화면 반영 후 API 저장, 실패 시 재로드
