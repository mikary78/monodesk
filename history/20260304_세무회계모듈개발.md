## 변경한 내용

### 추가된 파일 — Backend

| 파일 | 역할 |
|------|------|
| `backend/database.py` | SQLite 연결, 세션 관리, 테이블 생성 |
| `backend/main.py` | FastAPI 앱 엔트리포인트, CORS, 라우터 등록 |
| `backend/models/accounting.py` | SQLAlchemy 모델 3개 (ExpenseCategory, ExpenseRecord, SalesRecord) |
| `backend/schemas/accounting.py` | Pydantic 스키마 (요청/응답 유효성 검사) |
| `backend/services/accounting_service.py` | 비즈니스 로직 (CRUD, 손익 계산, 기본 데이터 초기화) |
| `backend/routers/accounting.py` | REST API 엔드포인트 전체 (prefix: /api/accounting) |

| 파일 | 역할 |

| `frontend/src/pages/AccountingPage.jsx` | 세무/회계 메인 페이지 (탭 네비게이션) |

## 데이터베이스 구조

### 신규 테이블

```sql
-- 지출 분류
expense_categories (id, name, description, color, is_deleted, created_at, updated_at)

-- 지출 기록
expense_records (id, expense_date, category_id, vendor, description,
                 amount, vat, payment_method, memo, receipt_image_path,
                 is_deleted, created_at, updated_at)

-- 매출 기록
sales_records (id, sales_date, cash_amount, card_amount, delivery_amount,
               memo, is_pos_synced, is_deleted, created_at, updated_at)
```

### 기본 지출 분류 데이터 (seed)
식재료비, 인건비, 임대료, 수도/광열비, 소모품비, 마케팅비, 기타

---

## API 엔드포인트 목록

```
GET    /api/accounting/categories          지출 분류 목록
POST   /api/accounting/categories          지출 분류 생성
PUT    /api/accounting/categories/{id}     지출 분류 수정
DELETE /api/accounting/categories/{id}     지출 분류 삭제

GET    /api/accounting/expenses            월별 지출 목록 (year, month 쿼리 파라미터)
GET    /api/accounting/expenses/{id}       지출 단건 조회
POST   /api/accounting/expenses            지출 생성
PUT    /api/accounting/expenses/{id}       지출 수정
DELETE /api/accounting/expenses/{id}       지출 삭제

GET    /api/accounting/sales               월별 매출 목록
POST   /api/accounting/sales               매출 생성

GET    /api/accounting/profit-loss         손익 계산 (year, month)
POST   /api/accounting/seed-categories     기본 분류 초기화
```

## 테스트 및 검증 방법

### Backend 테스트 실행
```bash
cd backend
pip install pytest httpx
pytest tests/test_accounting.py -v
```

### Frontend 테스트 실행
```bash
cd frontend
npm install
npx vitest run src/__tests__/accounting.test.jsx
```

### 수동 확인 사항
- [ ] 백엔드 서버 실행: `uvicorn main:app --reload --port 8000`
- [ ] API 문서 확인: http://localhost:8000/docs
- [ ] 기본 분류 초기화: POST /api/accounting/seed-categories 1회 실행
- [ ] 지출 입력 → 목록 조회 → 수정 → 삭제 플로우 확인
- [ ] 매출 입력 후 손익 계산 결과 확인
- [ ] CSV 내보내기 파일 생성 확인

## 주의사항

1. **최초 실행 시 기본 분류 초기화 필요**
   - 앱 첫 실행 후 `/api/accounting/seed-categories` POST 요청 1회 필요
   - 또는 프론트엔드에서 자동 호출 로직 추가 권장

2. **Python 의존성 설치 필요**
   ```bash
   pip install fastapi uvicorn sqlalchemy pydantic pytest httpx
   ```

3. **Frontend 의존성 설치 필요**
   ```bash
   npm install lucide-react @testing-library/react @testing-library/jest-dom vitest
   ```

4. **아직 연동되지 않은 기능**
   - 사이드바 네비게이션 (AccountingPage 라우팅)
   - 영수증 이미지 업로드 (receipt_image_path 필드 준비됨)
   - POS CSV 자동 가져오기
