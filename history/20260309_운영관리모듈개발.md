# 운영 관리 모듈 개발 이력 (Step 8 — 마지막 모듈)

## 작업 일시
2026-03-09

## 개요
MonoDesk 8개 모듈 중 마지막인 운영 관리 모듈을 개발했습니다.
제철해산물 주점(여남동)의 일상 운영에 필요한 기능을 4개 탭으로 구성했습니다.

---

## 구현 범위

### 탭 1 — 공지사항/메모
- 공지 유형: 공지(notice) / 메모(memo) / 긴급(urgent)
- 긴급 공지 자동 상단 고정
- 유형 필터 + 최신순/고정 우선 정렬
- CRUD 완전 구현 (소프트 삭제)

### 탭 2 — 위생 점검
- 체크리스트 항목 관리 (개점/마감/일상 × 주방/홀/화장실/설비)
- 날짜별 점검 결과 저장 (양호/불량/해당없음, 클릭 즉시 upsert)
- 당일 점검 완료율 통계 표시
- 기본 20개 항목 시드 (제철해산물 주점 특화)

### 탭 3 — 영업일 관리
- 월별 캘린더 UI — 날짜 클릭으로 우측 패널 편집
- 영업 상태: 정상영업 / 휴무 / 특별영업
- 날씨, 매출 목표, 특이사항 메모 기록
- 날짜별 upsert 방식 (UNIQUE: business_date)
- 월간 통계 (정상/휴무/특별 집계)

### 탭 4 — 업무 체크리스트
- 업무 구분: 개점 / 마감 / 주간 / 월간
- 항목 클릭으로 완료/미완료 즉시 토글 (낙관적 UI 업데이트)
- 전체 완료 / 전체 초기화 버튼
- 완료율 실시간 표시 (색상: 100%=초록, 70%+=파랑, 40%+=노랑, 미만=빨강)
- 기본 18개 항목 시드 (개점 6 / 마감 5 / 주간 4 / 월간 3)

## 설계 특이사항

### 정적 경로 우선 등록 원칙 준수
- `GET /hygiene/monthly-summary` — `GET /hygiene/records` 이전에 등록
- `GET /business-days/monthly-stats` — `GET /business-days/{date}` 이전에 등록
- `GET /tasks/records` — `GET /tasks/checklists/{task_id}` 이전에 등록

### upsert 패턴
- `hygiene_records`: (check_date, checklist_id) 조합 upsert
- `business_days`: business_date 기준 upsert
- `task_records`: (record_date, task_id) 조합 upsert

### 초기 시드 (멱등성 보장)
- `seed_default_hygiene_checklists`: 기존 항목 있으면 건너뜀
- `seed_default_task_checklists`: 기존 항목 있으면 건너뜀
